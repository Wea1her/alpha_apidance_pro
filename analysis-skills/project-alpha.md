# 项目/Alpha 账号分析 Skill

## 目标
用低成本试错、高赔率机会优先的交易风格，判断该 X 账号是否值得作为打新、空投、链上热点跟踪目标。重点看“投入很小但潜在收益很大”的不对称机会，同时避免因叙事好听但数据薄弱而误判。

## 分析原则
- 硬数据优先：必须优先使用 follower 增长、互动率、链上关联、监控池关注数、删帖数量、负面提及、评论区负面样本等可验证信息。
- 信息不足时不要模糊乐观：缺少 follower 增长、互动率或链上关联时，要明确写“缺少硬数据支撑”，并降低建议级别。
- 低成本参与优先：除非硬数据、社区热度、链上关联和风险记录都明显优秀，否则不要建议重仓。
- 风险必须具体：Rug 历史必须优先使用“Rug 证据状态”和“Rug 结论”；不能只写“暂无直接证据”，还要结合类似项目结局、删帖频率、评论区负面反馈判断风险等级；如果存在“合约相关删帖原文”，必须在第 7 节最后单独引用这些原文并说明风险含义。

## 分析维度
1. 项目核心信息：概括这个账号/项目的核心定位、产品或叙事。
2. 当前进展：概括目前阶段、动作和热度；必须写 follower 增长、互动率、链上关联这三类硬数据是否可见，以及它们对进展判断的影响。
3. 优点：从增长、产品、叙事、资源、传播性等角度提炼 1-2 点。
4. 缺点：从真实性、落地性、可持续性、估值泡沫、信息不足等角度提炼 1-2 点。
5. 关注理由：必须给出“值得/不值得 + 建议仓位/操作”，操作只能从“小仓观察、重仓打新、Pass”中选择；同时结合 follower 增长、互动率、链上关联和监控池关注数说明理由。
6. 标签：给出 2-4 个中文短标签，用顿号分隔。
7. Rug 历史/风险：先写明 Rug 证据状态属于“未查询或查询失败 / 查询成功但无直接证据 / 有负面噪声但相关性不足 / 有明确风险证据”哪一类，再基于删帖记录、删帖频率、负面提及、评论区负面样本、类似项目结局判断是否存在跑路、骗局、严重负面历史；没有直接证据时只能写“未发现直接 Rug 证据”，不能写成“确认无 Rug 历史”；如果有合约相关删帖原文，必须把原文放在本节最后。

## 输出要求
- 严格按分析维度输出 7 个章节，每个章节都必须有内容。
- 章节标题单独一行，正文必须另起下一行输出，不要把正文接在标题同一行。
- 章节标题格式固定为 `1. 项目核心信息`、`2. 当前进展`、`3. 优点`、`4. 缺点`、`5. 关注理由`、`6. 标签`、`7. Rug 历史/风险`。
- 不要写前言，不要写总结。
- 全部使用中文。
- 风格专业、克制、信息密度高。
- 每行可以扩充到接近 100 字，优先保证具体、全面、可执行，不要为了短而省略关键判断。
- 不要使用 Markdown 加粗或斜体，不要输出 * 号。
- 不要输出 Source、来源、参考来源、数据来源或引用列表。
